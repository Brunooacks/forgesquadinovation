# Definition of Ready (DoR)

Uma user story só entra em sprint quando TODOS os critérios abaixo são atendidos:

## Requisitos
- [ ] User story escrita no formato "As a... I want... So that..."
- [ ] Critérios de aceite definidos no formato Given/When/Then
- [ ] Requisitos não-funcionais identificados (performance, segurança, etc.)
- [ ] Dependências mapeadas e resolvidas (ou plano de resolução)

## Design
- [ ] Wireframe ou mockup aprovado pelo stakeholder (se aplicável)
- [ ] Arquitetura definida e validada pelo Arquiteto
- [ ] Contratos de API definidos (se aplicável)
- [ ] Modelo de dados revisado (se aplicável)

## Estimativa
- [ ] Story estimada pelo time (story points ou T-shirt sizing)
- [ ] Complexidade compreendida por pelo menos 2 membros do time
- [ ] Riscos técnicos identificados

## Testabilidade
- [ ] Critérios de aceite são testáveis automaticamente
- [ ] Cenários de teste de alto nível definidos pelo QA
- [ ] Dados de teste identificados ou planejados
