Estrutura de Dados

Como os Agentes Funcionam

Anatomia de um Agente

# agents/architect.agent.md
name: Arquiteto
role: System Architect
model_tier: opus (decisoes criticas)
participates_in: ALL phases
tools: [web_search, code_analysis]
persona:
  style: pragmatico, data-driven
  priorities:
    - qualidade arquitetural
    - trade-off analysis
    - ADR documentation
approval_required: true
outputs:
  - system_design.md
  - adr-001.md ... adr-N.md

Modos de Execucao

  • 1Persona Switching (Inline)
    O mesmo LLM assume diferentes personas sequencialmente dentro de uma unica sessao. Ideal para handoffs rapidos entre agentes.
  • 2Subagent Mode (Background)
    Cada agente roda em processo separado com contexto isolado. Ideal para tarefas longas como implementacao e testes.
  • 3Model Tier Routing
    Agentes criticos (Arquiteto, Finance) usam Opus. Tarefas operacionais usam Sonnet. Tasks simples usam Haiku. Otimiza custo sem perder qualidade.

Analogia: Pense em cada agente como um microsservico — interface definida (inputs/outputs), responsabilidade unica, comunicacao via contrato (YAML).

Pipeline Runner — Mecanica Interna

Fluxo de Execucao

// pipeline.yaml simplificado
phases:
  - name: estimativa
    agent: finance-advisor
    checkpoint: true
    approval_gate: budget_approval
    outputs: [cost_estimate.md]

  - name: requisitos
    agents: [ba, architect]
    tools: [kiro]
    checkpoint: true
    on_reject: re-elicit
    outputs:
      - user_stories.md
      - acceptance_criteria.md

Mecanismo de Checkpoints

Agente A
→
APPROVAL GATE
→
Agente B
  • ✓Pause automático: Pipeline para e notifica o humano
  • ✓Approve / Reject / Modify: 3 opcoes para o revisor
  • ✓on_reject: Pipeline volta ao step anterior automaticamente
  • ✓Audit log: Decisao, timestamp, quem aprovou, hash do artefato

Importante: Nenhum artefato passa para a proxima fase sem aprovacao humana explicita. Isso garante governanca real, nao teatro.

Audit Trail — Rastreabilidade Total

Estrutura do Log

// audit-trail.json (append-only)
{
  "id": "evt_2026031700042",
  "timestamp": "2026-03-17T14:32:01Z",
  "phase": "architecture",
  "step": "step-05-system-design",
  "agent": "architect",
  "action": "artifact_created",
  "artifact": {
    "name": "system-design.md",
    "hash": "sha256:a3f8b2c...",
    "size": 14520
  },
  "approval": {
    "status": "approved",
    "reviewer": "bruno.oliveira",
    "timestamp": "2026-03-17T14:45:12Z"
  }
}

Garantias Tecnicas

  • 🔒Imutabilidade: Log append-only. Nenhum registro pode ser alterado ou deletado apos escrita. Qualquer tentativa gera alerta.
  • 🔐Integridade: Cada artefato recebe hash SHA-256 no momento da criacao. Hash e verificado em cada leitura subsequente.
  • 📋Rastreabilidade: Cada evento rastreia: quem fez, quando, qual agente, qual step, qual artefato, quem aprovou.
  • 🏦Compliance: Formato compativel com requisitos de auditoria Bacen, LGPD (retencao e acesso), PCI DSS (log de acesso).

Para devs: Pense no audit trail como um event sourcing — cada acao gera um evento imutavel. Voce pode reconstruir o estado completo de qualquer projeto a partir do log.

Skills Engine — Extensibilidade via Plugins

Como funciona uma Skill

# skills/devin/SKILL.md
name: Devin
type: development
capabilities:
  - autonomous_coding
  - bug_fixing
  - feature_implementation
trigger:
  when: agent needs code implementation
  agents: [dev-backend, dev-frontend]
interface:
  input: task description + context
  output: PR URL + diff summary
config:
  api_key: ${DEVIN_API_KEY}
  workspace: ${PROJECT_ROOT}

Arquitetura de Plugins

  • ▶Interface Padronizada: Toda skill tem input/output definido em YAML. O Pipeline Runner sabe como invocar qualquer skill sem conhecer detalhes internos.
  • ▶Hot-pluggable: Adicione ou remova skills sem alterar o pipeline. Basta criar a pasta em skills/ com o SKILL.md.
  • ▶Skill Composition: Skills podem chamar outras skills. Ex: QA pode usar Copilot para gerar testes e SonarQube para analisar cobertura.
  • ▶Fallback Chain: Se uma skill falha, o agente pode usar uma alternativa. Ex: se Devin nao esta disponivel, usa Copilot inline.

Para devs: Skills sao como adapters do padrao Hexagonal. O dominio (agentes) nao depende da implementacao (tools). Voce pode trocar Devin por Cursor sem mudar nenhum agente.
